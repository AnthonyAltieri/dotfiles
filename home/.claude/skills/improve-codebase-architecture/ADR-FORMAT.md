# ADR Format

Use this reference only after the user explicitly authorizes recording an architecture decision. Follow the repository's existing ADR location, numbering, filename, headings, and status vocabulary when present. Otherwise use `docs/adr/NNNN-short-title.md` and the fallback below.

```markdown
# <Decision title>

- Status: proposed | accepted | superseded
- Date: YYYY-MM-DD

## Context

<The observed pressure, constraints, and decision drivers.>

## Decision

<The chosen direction and its scope.>

## Consequences

<Benefits, costs, risks, and follow-up obligations.>
```

Record the user's load-bearing reason and the alternatives actually considered. Do not manufacture consensus, imply approval beyond the user's authorization, or rewrite unrelated decisions. Read the completed ADR back before returning its path.
